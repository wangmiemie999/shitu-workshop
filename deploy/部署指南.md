# ImageCraft 部署指南

从零把 ImageCraft 部署到云服务器的完整步骤。全程只需要复制粘贴命令。

## 前提

一台 Ubuntu 系统的云服务器（阿里云/腾讯云轻量应用服务器默认镜像即可）、一个域名、OpenRouter 的 API Key。

## 第一步：把代码传到服务器

在自己电脑上，进入项目所在目录，执行（把 IP 换成你的服务器公网 IP）：

```bash
scp -r imagecraft-main root@服务器IP:/opt/imagecraft
```

Windows 用户如果没有 scp，可以用云平台网页控制台的"上传文件"功能，或安装 WinSCP 拖拽上传。上传后登录服务器（云平台网页上点"远程连接"即可）。

## 第二步：运行一键部署脚本

在服务器上执行：

```bash
cd /opt/imagecraft
sudo bash deploy/setup.sh 你的域名.com
```

脚本会自动完成：安装 Node.js 20 → 注册系统服务（崩溃自动重启、开机自启）→ 安装配置 Nginx → 安装 HTTPS 证书工具 → 生成 5 个首批邀请码（会打印在屏幕上，记下来）。

## 第三步：填入生图密钥

```bash
nano /opt/imagecraft/.env
```

把 `OPENROUTER_API_KEY=` 后面填上你的密钥，保存退出（Ctrl+O 回车，Ctrl+X），然后重启服务：

```bash
systemctl restart imagecraft
```

## 第四步：测试

浏览器打开 `http://服务器公网IP`，应能看到 ImageCraft 首页。用邀请码注册一个账号，走完整流程生成一张真图——如果生图报"接口超时"，说明服务器到 OpenRouter 的线路不通，联系开发协助加中转。

注意：如果打不开，检查云平台控制台的**防火墙/安全组**是否放行了 80 和 443 端口（轻量服务器默认可能只开了 22）。

## 第五步：备案通过后

1. 到域名控制台把域名解析（A 记录）指向服务器 IP；
2. 申请 HTTPS 证书（免费，自动续期）：
   ```bash
   certbot --nginx -d 你的域名.com -d www.你的域名.com
   ```
3. 编辑 `public/index.html` 页脚处的备案号占位，填入下发的备案号；
4. `systemctl restart imagecraft`，正式上线。

## 日常运维速查

```bash
systemctl status imagecraft          # 服务状态
journalctl -u imagecraft -f          # 实时日志（Ctrl+C 退出）
systemctl restart imagecraft         # 重启服务
node tools/invites.js create 10 --quota 20 --note 第二批   # 生成邀请码
node tools/invites.js list           # 邀请码使用情况
node tools/invites.js users          # 注册用户与额度消耗
nano data/blocklist.txt              # 扩充违禁词（每行一个，1分钟内自动生效）
node tools/stats.js                  # 运营数据：每日注册/活跃/生图量/成本估算
node tools/stats.js 30               # 查看最近30天
```

## 生图接口连不通怎么办（OpenRouter 中转）

如果测试时生图报"接口超时"，说明服务器直连 OpenRouter 不通。解法是用 Cloudflare Worker 免费中转，`deploy/openrouter-relay.js` 文件头部有完整的 10 分钟部署步骤。部署好 Worker 后在 `.env` 里加两行（Worker 地址和中转口令），重启服务即可，其余一切不变。

## 数据备份

所有用户数据在 `data/store.json` 一个文件里，生成的图片在 `outputs/` 目录。定期备份：

```bash
cp /opt/imagecraft/data/store.json ~/store-backup-$(date +%F).json
```

## 更新代码

新版本代码上传覆盖后（注意**不要覆盖** `data/` 目录和 `.env` 文件），执行 `systemctl restart imagecraft` 即可。

## 面向公众运营的提示

开源仓库不包含《用户协议》《隐私政策》页面。如果你要把本项目部署为面向公众的在线服务，请根据当地法规自行添加相应协议页面，并在注册流程与页脚中恢复链接。
